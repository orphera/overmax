# Overmax v0.2.2 릴리즈 노트

> v0.2.1 이후 변경 사항

---

## 🚀 주요 신규 기능 및 변경 사항

### ⚡ 전체화면 시 DXGI 기반 캡처 성능 개선 (Adaptive Capture Engine)
- 게임이 전체화면(Borderless Fullscreen 포함)으로 구동 중일 때, GDI 캡처 방식 외에 GPU 하드웨어 가속을 활용하는 DXGI Desktop Duplication 캡처 엔진을 작동시켜 CPU 점유율을 대폭 낮추고 실시간 분석 지연을 최소화했습니다. (일반 창 모드에서는 기존 GDI 엔진이 작동하며 불필요한 DXGI 리소스는 즉시 해제됩니다.)
- DXGI 엔진 오류(Access Lost 등) 발생 시 GDI 엔진으로 런타임에 투명하게 복구/롤백하는 `AdaptiveCaptureEngine` Facade 구조로 개편하여 캡처 실패로 인한 오동작을 최소화하고 안정성을 극대화하였습니다.

### 🖥️ 보더리스(전체 창 모드) 오버레이 최상위 Z-Order 및 포커스 유지
- 오버레이 창을 드래그해서 위치를 변경할 때 게임 화면이 최소화되거나 포커스를 빼앗기지 않도록 `WS_EX_NOACTIVATE` 스타일을 적용한 절대 드래그 감지 루틴을 설계했습니다.
- 포커스가 게임으로 복귀해도 오버레이가 항상 상위에 고정되도록 게임 창을 오버레이 창의 Owner (`GWL_HWNDPARENT`)로 지정하여 Z-Order 밀림 현상을 해결했습니다.
- 주기적인 윈도우 감시(1초 주기 하트비트)를 통해 `HWND_TOPMOST` 상태를 유지시킵니다.

### 📱 라이트 모드 (Lite Mode) 및 오버레이 스냅(고정 위치) 기능 도입
- **라이트 모드**: 추천 리스트 등 불필요한 레이아웃을 숨기고 곡 제목, 난이도 정보, 실시간 레이트 등 핵심 정보만 보여주는 콤팩트한 요약 레이아웃(세로 높이 약 60px 수준)을 제공합니다.
- **오버레이 스냅 (고정 위치)**: 라이트 모드 여부와 관계없이 오버레이를 화면 구석(좌상단, 우상단, 좌하단, 우하단)으로 흔들림 없이(Jitter-free) 자동 밀착 스냅 고정시킬 수 있는 기능입니다.
- **수동 기본값 및 드래그 제어**: 스냅 위치의 기본값은 **"수동" (`manual`)**으로 설정되어 있어 기존처럼 원하는 좌표에 자유롭게 드래그하여 배치할 수 있습니다. 반면, 스냅이 활성화된 상태(수동이 아님)일 때는 일반/라이트 모드와 무관하게 드래그가 잠겨 화면에서 어긋나지 않도록 보호합니다.
- **시각적인 설정 UI**: 설정 창의 고정 위치 지정을 위한 버튼 배치가 모니터 가상 프레임(검은색 직사각형) 내부 모퉁이에 직관적으로 매핑되는 미학적인 레이아웃으로 변경되었습니다.

### 🛠️ 자가 업데이트 후 자동 재시작 락(Mutex) 충돌 버그 해결
- 자가 업데이트 성공 후 새 버전 실행 시 기존 프로세스가 들고 있던 단일 실행 락(`OvermaxSingleInstanceMutex`)이 미처 해제되지 않아 새 인스턴스가 조기 종료되던 버그를 해결했습니다.
- 새 프로세스를 실행(spawn)하기 전 부모 프로세스의 단일 실행 락 가드(`SingleInstanceGuard`)를 명시적으로 `drop`하도록 워크플로우를 개편하여 매끄러운 자가 업데이트가 진행되도록 하였습니다.

### 🔍 기타 수정 사항
- 업데이트 확인 알림창의 버전 정보에서 `v` 접두사가 중복 표시되던 오타를 수정하였습니다.
- 마우스 호버 여부에 따른 투명도 반응 지연 및 렌더링 깜빡임 현상을 크게 개선하였습니다.
