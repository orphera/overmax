# Overmax v0.2.3 릴리즈 노트

> v0.2.2 이후 변경 사항

---

## 🚀 주요 신규 기능 및 변경 사항

### ⚡ Topmost 윈도우 제어 및 포커스 감시 최적화
* **포커스 감시 및 z-order 꼬임 현상 해결**:
  * 기존에 1초 주기 하트비트 타이머에 의해 강제되고 매 프레임 실행되던 활성 포커스 감시(`is_active` 판별) 로직을 개선했습니다.
  * 특히 비활성 상태(`is_active == false`)일 때 topmost 스타일 검증 오류로 인해 매 프레임 `SetWindowPos(HWND_NOTOPMOST)`가 무한히 스팸 호출되던 캐시 미스 버그를 수정하여, 게임 외의 다른 창이 활성화될 때 z-order가 꼬이거나 깜빡이는 오동작을 완벽하게 예방했습니다.
* **게임 윈도우 핸들 캐싱**:
  * 한 번 찾은 게임 창 핸들을 구조체 필드로 캐싱하고 `IsWindow` 유효성 검사 API만 활용해 생존 여부를 체크하도록 하여, 매 프레임 `FindWindowW`가 불리던 오버헤드를 완전 차단(평상시 API 호출 횟수 **0회**)했습니다.

### 🖱️ egui 네이티브 뷰포트 드래그 도입 (`StartDrag`)
* **불필요한 마우스 좌표 계산 연산 제거**:
  * 마우스 드래그를 처리할 때 기존에 수동으로 계산하던 마우스 절대 좌표 및 배율(`ppi`) 역산 연산을 모두 걷어냈습니다.
  * egui의 공식 표준 뷰포트 드래그 위임 API인 `ViewportCommand::StartDrag`를 사용하여 OS 레벨에서 윈도우 드래그를 네이티브로 안전하게 처리하도록 변경했습니다.
  * 이로써 렉 없이 부드러운 드래그 이동을 실현하고, 구조체에 불필요하게 유지되던 마우스 상태 멤버 변수들을 완전히 삭제하여 코드를 슬림화했습니다.

### 🖥️ 구석 고정 (Snap Position) 최적화 및 락 경합 해제
* **비동기 락 획득 (`try_lock`)**:
  * 매 프레임 UI 스레드에서 구석 정렬을 체크하기 위해 동기식으로 획득되던 `game_rect` 락을 `try_lock()`으로 교체하여, 백그라운드 캡처 스레드와의 락 경합으로 인한 메인 UI 스터터링(프레임 드랍) 가능성을 원천 차단했습니다.
* **불필요한 SetWindowPos 호출 제거**:
  * 오버레이가 구석에 고정되어 있는 평상시에도 매 프레임 `SetWindowPos`를 날리던 무조건적인 갱신 로직을 수정했습니다.
  * 이전 고정 위치/크기 기하 캐시(`PREV_SNAP_GEOMETRY`)와 비교하여 좌표나 크기가 변경되었을 때만 딱 1번 `SetWindowPos`를 호출하도록 개선하여 API 호출 횟수를 **0회**로 최적화했습니다.

### 🛠️ 보조 창 수명 주기 및 포커스(Bring to Front) UX 개선
* **종료 시 보조 창 순차적 조기 정리**:
  * 트레이에서 "종료" 클릭 시, 메인 창을 바로 닫기 전 켜져 있는 설정, 동기화, 디버그 창의 상태를 안전하게 `false`로 먼저 변경하여 뷰포트 종료 자원 정리와 Draft 세팅 저장이 비정상 종료 없이 매끄럽게 마무리되도록 보장했습니다.
* **이미 열려 있는 창의 포커스 복구**:
  * 이미 켜져 있는 설정, 동기화, 디버그 창이 다른 창 뒤에 가려져 보이지 않을 때, 트레이 아이콘 메뉴에서 해당 메뉴를 다시 클릭하면 `ViewportCommand::Focus`를 날려 화면 맨 앞으로(Bring to Front) 즉시 띄워 주도록 개선했습니다.

### 🌍 마우스 오버 감지 크로스 플랫폼 이식성 확보
* **target_os 조건부 컴파일 가드 추가**:
  * 마우스 클릭 관통 모드 동작을 보장하기 위해 전역 마우스 좌표(`GetCursorPos`) 감시가 필수적인 `is_mouse_over_overlay` 함수에 `#[cfg(target_os = "windows")]` 가드를 세워 Windows 환경이 아닌 곳(macOS, Linux)에서 컴파일이 깨지는 현상을 방지하고, egui 내장 API 기반 fallback 처리를 마련했습니다.
